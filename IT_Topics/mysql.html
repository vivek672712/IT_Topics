What is SQL?

SQL (Structured Query Language) is a standard language for managing and manipulating databases. It is widely used to perform operations such as querying, updating, and managing data in relational database management systems (RDBMS).

                    SELECT * FROM users WHERE age > 25;
            -- This is a comment
            INSERT INTO users (name, age) VALUES ('Alice', 30);

SQL Introduction

History of SQL

SQL, or Structured Query Language, was initially developed at IBM by Donald D. Chamberlin and Raymond F. Boyce in the early 1970s. It was designed for managing and manipulating data in relational database management systems (RDBMS). The first commercial implementation of SQL was released by Oracle in 1979.

Introduction to SQL

SQL is a standard language used to interact with databases. It is designed to manage and manipulate relational databases by providing a way to query, update, insert, and delete data. SQL is widely used in data management, business intelligence, and web development, making it an essential tool for database administrators and developers.

How SQL Works

SQL operates by sending commands to a database, which are then executed by the database management system (DBMS). SQL statements can be used to create and modify database structures, insert and update data, and retrieve data from one or more tables. SQL's declarative nature allows users to specify what data they want to retrieve or manipulate, without needing to know the underlying procedures to accomplish these tasks.

SQL Syntax

SQL Statements

SQL statements are the commands used to communicate with a database. They typically begin with a keyword such as SELECT, INSERT, UPDATE, or DELETE, and are used to perform various operations on the database.

                    SELECT * FROM users WHERE age > 30;

In this example, the SQL statement retrieves all records from the users table where the age column is greater than 30.

SQL Comments

Comments in SQL are used to describe what the SQL statements do and are ignored by the SQL engine. Single-line comments are marked with --, while multi-line comments are enclosed between /* and */.

                    -- This is a single-line comment
            /* This is a multi-line comment
               that spans multiple lines */
            SELECT * FROM users WHERE age > 30;

The -- starts a single-line comment, and everything following it on the same line is ignored. Multi-line comments are placed between /* and */ to span across multiple lines.

SQL SELECT Statement

Basic SELECT Statement

The SELECT statement in SQL is used to retrieve data from a database. The data returned is stored in a result table, often referred to as the result set.

                    SELECT column1, column2 FROM table_name;

In this example, the SELECT statement retrieves the specified columns column1 and column2 from the table table_name.

SELECT * Statement

The SELECT * statement is used to select all columns from a table.

                    SELECT * FROM users;

This statement retrieves all columns from the users table.

WHERE Clause

The WHERE clause is used to filter records that meet certain conditions.

                    SELECT * FROM users WHERE age > 30;

In this example, the SELECT statement retrieves all records from the users table where the age column is greater than 30.

SQL SELECT DISTINCT Statement

The SELECT DISTINCT statement in SQL is used to retrieve unique values from a column in a table. It eliminates duplicate records from the result set.

                    
            SELECT DISTINCT column_name FROM table_name;

For example, if you have a table named employees with a column department, the following query retrieves the unique departments:

                    
            SELECT DISTINCT department FROM employees;

This statement returns all unique department names from the employees table.

SQL WHERE Clause

The WHERE clause in SQL is used to filter records in a query based on specific conditions. It allows you to extract only those records that fulfill the given condition(s).

                    SELECT * FROM employees
            WHERE department = 'Sales';

In this example, the WHERE clause filters the records from the employees table to only include those where the department is 'Sales'.

SQL ORDER BY Clause

The ORDER BY clause in SQL is used to sort the result set in ascending or descending order based on one or more columns.

                    
            -- Sort by a single column in ascending order
            SELECT * FROM employees
            ORDER BY salary;
            
            -- Sort by a single column in descending order
            SELECT * FROM employees
            ORDER BY salary DESC;
            
            -- Sort by multiple columns (e.g., department ascending, salary descending)
            SELECT * FROM employees
            ORDER BY department, salary DESC;

In these examples, the ORDER BY clause sorts the records from the employees table by the salary column in ascending or descending order. You can also sort by multiple columns, as shown in the last example.

SQL AND Operator

Using the AND Operator in SQL

The AND operator in SQL is used to combine multiple conditions in a WHERE clause. All conditions must be true for a record to be included in the result set.

                    
            -- Retrieve employees who work in the 'Sales' department and have a salary greater than 50000
            SELECT * FROM employees
            WHERE department = 'Sales'
            AND salary > 50000;

In this example, the query returns records from the employees table where both conditions are met: the employee works in the 'Sales' department and has a salary greater than 50,000.

Combining Multiple Conditions with AND

You can use the AND operator to combine more than two conditions in a SQL query.

                        
            -- Retrieve employees who work in 'Sales', have a salary greater than 50000, and have more than 5 years of experience
            SELECT * FROM employees
            WHERE department = 'Sales'
            AND salary > 50000
            AND experience > 5;

This query returns records from the employees table where all three conditions are true: the employee works in 'Sales', has a salary greater than 50,000, and has more than 5 years of experience.

SQL OR Operator

The OR operator in SQL is used to combine multiple conditions in a WHERE clause. If any of the conditions are true, the record will be included in the result set.

Basic OR Operator Example

                    
            -- Retrieve employees who work in the 'Sales' department or have a salary greater than 50000
            SELECT * FROM employees
            WHERE department = 'Sales'
            OR salary > 50000;

Output: This query returns records from the employees table where either the employee works in the 'Sales' department or has a salary greater than 50,000.

OR Operator with Multiple Conditions

                    
            -- Retrieve employees who work in 'Sales' or 'Marketing' or have a salary greater than 50000
            SELECT * FROM employees
            WHERE department = 'Sales'
            OR department = 'Marketing'
            OR salary > 50000;

Output: This query returns records from the employees table where the employee works in either 'Sales' or 'Marketing', or has a salary greater than 50,000.

Combining AND with OR

                    
            -- Retrieve employees who work in 'Sales' and have more than 5 years of experience, or have a salary greater than 50000
            SELECT * FROM employees
            WHERE (department = 'Sales' AND experience > 5)
            OR salary > 50000;

Output: This query returns records where the employee works in 'Sales' and has more than 5 years of experience, or where the employee has a salary greater than 50,000.

SQL NOT Operator

The NOT operator in SQL is used to negate a condition in a WHERE clause. It returns records that do not satisfy the specified condition.

Basic NOT Operator Example

                    
            -- Retrieve employees who do not work in the 'Sales' department
            SELECT * FROM employees
            WHERE NOT department = 'Sales';

Output: This query returns records from the employees table where the department is not 'Sales'.

NOT Operator with Comparison

                    
            -- Retrieve employees whose salary is not greater than 50000
            SELECT * FROM employees
            WHERE NOT salary > 50000;

Output: This query returns records where the salary is 50,000 or less.

Combining NOT with Other Operators

                    
            -- Retrieve employees who do not work in 'Sales' or 'Marketing'
            SELECT * FROM employees
            WHERE NOT (department = 'Sales' OR department = 'Marketing');

Output: This query returns records from the employees table where the department is neither 'Sales' nor 'Marketing'.

NOT Operator with Multiple Conditions

                    
            -- Retrieve employees who do not work in 'Sales' and do not have a salary greater than 50000
            SELECT * FROM employees
            WHERE NOT (department = 'Sales' AND salary > 50000);

Output: This query returns records where either the department is not 'Sales' or the salary is 50,000 or less, or both.

SQL INSERT INTO Statement

The INSERT INTO statement is used to add new rows to a table in SQL. You can insert data into one or more columns of a table.

Basic INSERT INTO Example

                    
            -- Insert a single row into the employees table
            INSERT INTO employees (name, department, salary)
            VALUES ('John Doe', 'Sales', 60000);

Output: This statement inserts a new row into the employees table with the name 'John Doe', department 'Sales', and salary 60,000.

INSERT INTO with Multiple Rows

                    
            -- Insert multiple rows into the employees table
            INSERT INTO employees (name, department, salary)
            VALUES 
                ('Jane Smith', 'Marketing', 70000),
                ('Alice Johnson', 'HR', 55000),
                ('Bob Brown', 'IT', 80000);

Output: This statement inserts multiple rows into the employees table with the specified names, departments, and salaries.

INSERT INTO with Default Values

                    
            -- Insert a row into the employees table using default values
            INSERT INTO employees (name, department)
            VALUES ('Sam Green', 'Finance');

Output: This statement inserts a new row into the employees table with the name 'Sam Green' and department 'Finance', using default values for any unspecified columns.

INSERT INTO with Subquery

                    
            -- Insert data from one table into another
            INSERT INTO new_employees (name, department, salary)
            SELECT name, department, salary
            FROM employees
            WHERE salary > 60000;

Output: This statement inserts rows into the new_employees table by selecting rows from the employees table where the salary is greater than 60,000.

SQL NULL Values

In SQL, NULL represents the absence of a value or an unknown value. It is used to indicate that a particular field does not have a value or that the value is unknown.

Inserting NULL Values

                    
            -- Inserting a row with NULL values
            INSERT INTO employees (name, department, salary)
            VALUES ('John Doe', 'Sales', NULL);

Output: This statement inserts a new row into the employees table with 'John Doe' in the name and department columns, and NULL in the salary column.

Querying NULL Values

                    
            -- Query to find rows where salary is NULL
            SELECT * FROM employees
            WHERE salary IS NULL;

Output: This statement selects all rows from the employees table where the salary column contains NULL.

Updating NULL Values

                    
            -- Updating a row to set salary to NULL
            UPDATE employees
            SET salary = NULL
            WHERE name = 'Jane Smith';

Output: This statement updates the salary of 'Jane Smith' to NULL in the employees table.

NULL Values in Conditions

                    
            -- Checking if a column is NOT NULL
            SELECT * FROM employees
            WHERE salary IS NOT NULL;

Output: This statement selects all rows from the employees table where the salary column does not contain NULL.

SQL UPDATE Statement

The UPDATE statement in SQL is used to modify the existing records in a table. You can specify which records to update and the new values using this statement.

Basic UPDATE Statement

                    
            -- Updating a single column in a table
            UPDATE employees
            SET salary = 50000
            WHERE name = 'John Doe';

Output: This statement updates the salary column to 50000 for the row where the name is 'John Doe'.

Updating Multiple Columns

                    
            -- Updating multiple columns in a table
            UPDATE employees
            SET department = 'Marketing', salary = 60000
            WHERE name = 'Jane Smith';

Output: This statement updates both the department and salary columns for the row where the name is 'Jane Smith'.

Updating All Rows

                    
            -- Updating all rows in a table
            UPDATE employees
            SET status = 'Active';

Output: This statement sets the status column to 'Active' for all rows in the employees table.

Conditional Updates

                    
            -- Updating rows based on a condition
            UPDATE employees
            SET salary = salary * 1.10
            WHERE department = 'Sales';

Output: This statement increases the salary by 10% for all employees in the 'Sales' department.

Using Subqueries in UPDATE

                    
            -- Updating based on a subquery
            UPDATE employees
            SET salary = (
                SELECT AVG(salary)
                FROM employees
                WHERE department = 'HR'
            )
            WHERE department = 'Marketing';

Output: This statement sets the salary of all employees in the 'Marketing' department to the average salary of employees in the 'HR' department.

SQL DELETE Statement

The DELETE statement in SQL is used to remove existing records from a table. You can specify which records to delete using a condition.

Basic DELETE Statement

                    
            -- Deleting a single record from a table
            DELETE FROM employees
            WHERE name = 'John Doe';

Output: This statement deletes the row where the name is 'John Doe' from the employees table.

Deleting Multiple Records

                    
            -- Deleting multiple records from a table
            DELETE FROM employees
            WHERE department = 'Marketing';

Output: This statement deletes all rows where the department is 'Marketing' from the employees table.

Deleting All Rows

                    
            -- Deleting all rows from a table
            DELETE FROM employees;

Output: This statement deletes all rows from the employees table. The table structure remains intact.

Conditional Deletion

                    
            -- Deleting rows based on a condition
            DELETE FROM employees
            WHERE salary < 30000;

Output: This statement deletes all rows where the salary is less than 30000 from the employees table.

Using Subqueries in DELETE

                    
            -- Deleting rows based on a subquery
            DELETE FROM employees
            WHERE department IN (
                SELECT department
                FROM departments
                WHERE budget < 10000
            );

Output: This statement deletes rows from the employees table where the department is in the list of departments with a budget less than 10000.

SQL TOP Clause

The TOP clause in SQL is used to specify the number or percentage of records to return in a query result. It is commonly used to limit the result set to a specified number of rows.

Basic TOP Clause


            -- Selecting the top 5 records from a table
            SELECT TOP 5 * 
            FROM employees;

Output: This query retrieves the first 5 rows from the employees table.

TOP with ORDER BY


            -- Selecting the top 3 highest salaries
            SELECT TOP 3 *
            FROM employees
            ORDER BY salary DESC;

Output: This query retrieves the top 3 employees with the highest salaries from the employees table.

TOP with Percentage


            -- Selecting the top 10% of records
            SELECT TOP 10 PERCENT *
            FROM employees;

Output: This query retrieves the top 10% of rows from the employees table.

Using TOP with Subqueries


            -- Selecting the top 5 records from a subquery
            SELECT *
            FROM (
                SELECT TOP 10 * 
                FROM employees
                ORDER BY salary DESC
            ) AS top_10_salaries
            ORDER BY name;

Output: This query first selects the top 10 highest salaries and then retrieves the top 5 records from that subset, ordered by name.

SQL Aggregate Functions

SQL aggregate functions perform a calculation on a set of values and return a single value. They are often used with the GROUP BY clause to aggregate results based on certain criteria.

  • SUM: Calculates the total sum of a numeric column.
  • AVG: Calculates the average value of a numeric column.
  • COUNT: Counts the number of rows in a result set or the number of non-NULL values in a column.
  • MIN: Finds the smallest value in a column.
  • MAX: Finds the largest value in a column.

The general syntax for SQL aggregate functions is:


            SELECT AGGREGATE_FUNCTION(column_name)
            FROM table_name
            WHERE condition;

Example of SQL Aggregate Functions


            -- Example of SUM function
            SELECT SUM(salary) AS total_salaries
            FROM employees;
            
            -- Example of AVG function
            SELECT AVG(salary) AS average_salary
            FROM employees;
            
            -- Example of COUNT function
            SELECT COUNT(*) AS number_of_employees
            FROM employees;
            
            -- Example of MIN function
            SELECT MIN(salary) AS lowest_salary
            FROM employees;
            
            -- Example of MAX function
            SELECT MAX(salary) AS highest_salary
            FROM employees;

Output: - The SUM function returns the total sum of the salary column. - The AVG function returns the average salary. - The COUNT function returns the total number of rows in the employees table. - The MIN function returns the lowest salary. - The MAX function returns the highest salary.

SQL MIN() and MAX() Functions

The MIN() and MAX() functions in SQL are used to find the smallest and largest values in a numeric column, respectively. These functions are useful for performing statistical operations on your data.

  • MIN(): Returns the smallest value in a specified column.
  • MAX(): Returns the largest value in a specified column.

The general syntax for the MIN() and MAX() functions is:


            SELECT MIN(column_name) AS min_value
            FROM table_name
            WHERE condition;
            
            SELECT MAX(column_name) AS max_value
            FROM table_name
            WHERE condition;

Example of SQL MIN() and MAX() Functions


            -- Example of MIN() function
            SELECT MIN(salary) AS lowest_salary
            FROM employees;
            
            -- Example of MAX() function
            SELECT MAX(salary) AS highest_salary
            FROM employees;

Output: - The MIN() function returns the smallest value in the salary column. - The MAX() function returns the largest value in the salary column.

SQL COUNT() Function

The COUNT() function in SQL is used to count the number of rows in a database table or the number of rows that match a specified condition. It is commonly used for aggregation and data analysis.

  • COUNT(*): Counts all rows in a table, including rows with NULL values.
  • COUNT(column_name): Counts the number of non-NULL values in a specified column.

The general syntax for the COUNT() function is:


            SELECT COUNT(*)
            FROM table_name
            WHERE condition;
            
            SELECT COUNT(column_name)
            FROM table_name
            WHERE condition;

Example: Counting All Rows


            -- Example of COUNT() function to count all rows
            SELECT COUNT(*)
            FROM employees;

Example: Counting Non-NULL Values


            -- Example of COUNT() function to count non-NULL values in a specific column
            SELECT COUNT(salary)
            FROM employees;

Output: - The first query returns the total number of rows in the employees table. - The second query returns the number of non-NULL values in the salary column.

SQL SUM() Function

The SUM() function in SQL is used to calculate the total sum of a numeric column in a table. It is often used in aggregation queries to get the total of values in a column.

  • SUM(column_name): Calculates the total sum of the values in the specified column. It ignores NULL values.

The general syntax for the SUM() function is:


            SELECT SUM(column_name)
            FROM table_name
            WHERE condition;

Example: Calculating the Total Sales


            -- Example of SUM() function to calculate the total sales
            SELECT SUM(sales_amount)
            FROM sales;

Example: Calculating the Total Salary


            -- Example of SUM() function to calculate the total salary
            SELECT SUM(salary)
            FROM employees;

Output: - The first query returns the total of all values in the sales_amount column from the sales table. - The second query returns the total of all values in the salary column from the employees table.

SQL AVG() Function

The AVG() function in SQL is used to calculate the average value of a numeric column. It is often used in queries to obtain the mean value of a dataset.

  • AVG(column_name): Computes the average of the values in the specified column. It ignores NULL values.

The general syntax for the AVG() function is:


            SELECT AVG(column_name)
            FROM table_name
            WHERE condition;

Example: Calculating the Average Sales


            -- Example of AVG() function to calculate the average sales
            SELECT AVG(sales_amount)
            FROM sales;

Example: Calculating the Average Salary


            -- Example of AVG() function to calculate the average salary
            SELECT AVG(salary)
            FROM employees;

Output: - The first query returns the average of all values in the sales_amount column from the sales table. - The second query returns the average of all values in the salary column from the employees table.

SQL BETWEEN Operator

The BETWEEN operator in SQL is used to filter the results within a specified range. It can be used with numeric, date, and text values to select records that fall between two values.

  • Range: The BETWEEN operator includes the boundary values in the range.
  • Usage: It is commonly used in the WHERE clause to specify a range of values.

The general syntax for the BETWEEN operator is:


            SELECT column_name(s)
            FROM table_name
            WHERE column_name BETWEEN value1 AND value2;

Example: Selecting Records Between Two Dates


            -- Example of BETWEEN operator to filter records within a date range
            SELECT *
            FROM orders
            WHERE order_date BETWEEN '2024-01-01' AND '2024-12-31';

Example: Selecting Records Within a Numeric Range


            -- Example of BETWEEN operator to filter records within a numeric range
            SELECT *
            FROM products
            WHERE price BETWEEN 50 AND 100;

Output: - The first query retrieves all orders placed within the year 2024. - The second query retrieves all products with a price between 50 and 100, inclusive.

SQL Joins

SQL Joins are used to combine rows from two or more tables based on a related column between them. Joins are essential for querying data from multiple tables and are categorized into several types:

  • INNER JOIN: Returns records that have matching values in both tables.
  • LEFT JOIN (or LEFT OUTER JOIN): Returns all records from the left table and the matched records from the right table. The result is NULL on the right side when there is no match.
  • RIGHT JOIN (or RIGHT OUTER JOIN): Returns all records from the right table and the matched records from the left table. The result is NULL on the left side when there is no match.
  • FULL JOIN (or FULL OUTER JOIN): Returns all records when there is a match in either left or right table. It returns NULL for non-matching rows from both tables.
  • CROSS JOIN: Returns the Cartesian product of the two tables, i.e., all possible combinations of rows from both tables.

Example: INNER JOIN

The following query retrieves records with matching values in both employees and departments tables:


SELECT employees.name, departments.department_name
FROM employees
INNER JOIN departments
ON employees.department_id = departments.department_id;

Example: LEFT JOIN

This query retrieves all employees and their department names, including employees who do not belong to any department:


SELECT employees.name, departments.department_name
FROM employees
LEFT JOIN departments
ON employees.department_id = departments.department_id;

Example: RIGHT JOIN

This query retrieves all departments and their employees, including departments with no employees:


SELECT employees.name, departments.department_name
FROM employees
RIGHT JOIN departments
ON employees.department_id = departments.department_id;

Example: FULL JOIN

This query retrieves all employees and all departments, including records with no matches in either table:


SELECT employees.name, departments.department_name
FROM employees
FULL JOIN departments
ON employees.department_id = departments.department_id;

Example: CROSS JOIN

This query returns the Cartesian product of the colors and sizes tables:


SELECT colors.color_name, sizes.size_name
FROM colors
CROSS JOIN sizes;

Output: - The INNER JOIN example will show employees with their corresponding department names. - The LEFT JOIN example will include all employees, even those without a department. - The RIGHT JOIN example will include all departments, even those without employees. - The FULL JOIN example includes all employees and departments, showing NULLs where there are no matches. - The CROSS JOIN example shows every possible combination of color and size.
